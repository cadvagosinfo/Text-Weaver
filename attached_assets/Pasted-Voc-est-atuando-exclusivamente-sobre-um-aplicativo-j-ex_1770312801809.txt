OBJETIVO

Criar uma nova aba denominada “CARTORIAL”, integrada ao aplicativo existente, sem modificar qualquer outra funcionalidade, aba, lógica, layout, estilo global ou estrutura do sistema.

1. ESTRUTURA DA ABA CARTORIAL

A aba CARTORIAL deve ser adicionada ao sistema de navegação já existente, mantendo o padrão atual do aplicativo.

Ao ser acessada, a aba deve gerar automaticamente tabelas com base nos participantes cadastrados na aba RELEASE.

Deve ser criada uma tabela para cada participante existente na aba RELEASE.

Caso existam múltiplos participantes, as tabelas devem ser renderizadas sequencialmente, uma abaixo da outra, com espaçamento visual adequado.

As tabelas devem possuir dimensões fixas, sem possibilidade de redimensionamento pelo usuário.

O layout da tabela deve replicar fielmente o modelo fornecido, sem ajustes, adaptações ou variações.

2. DADOS COM PREENCHIMENTO AUTOMÁTICO (ORIGEM: ABA RELEASE)

Para cada participante da aba RELEASE, preencher automaticamente os seguintes campos, somente se houver dados disponíveis:

NOME: nome do envolvido

RG: resposta de RG

CPF: resposta de CPF

DN: data de nascimento

OC: resposta de antecedentes

ORCRIM: resposta de organização criminosa

Não criar dados fictícios.
Não alterar dados de origem.
